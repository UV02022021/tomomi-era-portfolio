# 🎯 Drag & Drop デプロイメントガイド

## 📁 **デプロイ用ファイル**

Drag & Dropでデプロイする場合、以下のファイルが必要です：

### ✅ デプロイするファイル一覧

```
dist/
├── index.html                    (609 bytes)
└── assets/
    ├── index-2d49bb99.js        (453,989 bytes - メインJavaScript)
    └── index-7364d70a.css       (35,952 bytes - スタイル)
```

## 🚀 **デプロイ方法**

### 方法1: distフォルダ全体をドラッグ (推奨)
1. `dist` フォルダ全体を選択
2. Vercel/Netlifyのサイトにドラッグ&ドロップ
3. 完了！

### 方法2: ファイル個別選択
以下の3つのファイル/フォルダを選択してドラッグ：
- ✅ `dist/index.html`
- ✅ `dist/assets/` フォルダ (中身含む)

## 🌐 **対応プラットフォーム**

### Vercel
1. [vercel.com](https://vercel.com) にアクセス
2. 「New Project」をクリック
3. `dist`フォルダをドラッグ&ドロップ
4. 「Deploy」をクリック

### Netlify
1. [netlify.com](https://netlify.com) にアクセス  
2. 「Sites」ページで`dist`フォルダをドラッグ&ドロップ
3. 自動でデプロイ開始

### Surge.sh
1. [surge.sh](https://surge.sh) にアクセス
2. `dist`フォルダをドラッグ&ドロップ
3. ドメイン名を入力
4. 「Publish」をクリック

## ⚠️ **重要な注意点**

### ❌ デプロイしてはいけないファイル：
- `src/` フォルダ (ソースコード)
- `node_modules/` フォルダ (依存関係)
- `package.json` (設定ファイル)
- `.git/` フォルダ (Gitデータ)

### ✅ デプロイするのは`dist/`フォルダのみ

## 🎯 **デプロイ後の確認項目**

デプロイ成功後、以下を確認してください：

- ✅ ダッシュボードが表示される
- ✅ プロジェクト一覧が正しく表示
- ✅ スキルページに遷移できる

## 🔄 **更新方法**

コードを変更した場合：

1. `npm run build` を実行
2. 新しい`dist/`フォルダを再度ドラッグ&ドロップ
3. 自動で更新される

## 🆘 **トラブルシューティング**

### 問題: ページが表示されない
**解決法**: `index.html`が含まれているか確認

### 問題: スタイルが適用されない  
**解決法**: `assets/`フォルダが含まれているか確認

### 問題: 404エラーが出る
**解決法**: SPA設定が必要。Vercelの場合は自動設定される
